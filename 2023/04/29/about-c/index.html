On one level, it can deal with files by using the basic file tools of the host operating system. This is called low-level I/O

C also deals with files on a second level called the standard I/O package

Conceptually, the C program deals with a stream instead of directly with a file.标准的文件包会在系统文件格式和程序需要处理的文件格式之间做转化，比如行结束符。所以c语言处理的是流而不是文件。

A stream is an idealized flow of data to which the actual input or output is mapped.流就是理性化的真正的文件所被处理，被映射出来的的数据组成的河流一样的东西（一个挨着一个，序列样式的）

c处理输入输出设备为文件，就想普通文件一样。处理键盘为stdin，屏幕输出为stdout. getchar(), putchar(), printf(), scanf()都是stdiopackage 里面的东西, 他们处理stdout和stdin这两个流

所以呢，流是通用的，c用来判断文件结束的结束符号同样可以用在设备上。引出了EOF。

EOF is defined in stdio.h
generally its -1 cause that doesn’t confilct with any ascii code(0-255).

most Unix and Linux systems, pressing Ctrl+D at the beginning of a line cause the end-of-line signal to be transmitted.

<,> |: first two is redirection, last is pipe. file to program, program to program.

Storage Classes, Linkage, and Memory Management

  1. storage classes: models used to storing data in memory.
  2. key consept

  • identifier: a way to access the object
  • storage duration: how long it stays in memory.
  • scope: block, function, function prototype, file scope
  • linkage: external, internal
  • storage Duration: static, thread, automatic, allocated
  1. storage classes:
